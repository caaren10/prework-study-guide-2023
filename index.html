HTML

  • HTML is made up of elements which provide context and basic framework for the webpage
  • There are different types of elements: text elements, image elements, header elements, and list elements
  • The head element contains information about the webpage. These contents are not visible when the user enters the webpage. Metadata helps the browser render the page correctly. This element is a container for metadata. This is content is used for Search Engine Optimization (SEO).
  • The body element represents the visible content shown to the user

CSS

  • CSS is an acronym for Cascading Style Sheets
  • CSS is used to define the colors, aesthetics, visual layout, and organization of a webpage
  • There are three ways to style using CSS: inline CSS, internal CSS, and external CSS
  • inline CSS and internal CSS are both used when a developer styles an element directly in HTML, and it is specific to the HTML tag
  • Inline CSS and Internal CSS are both useful for quick changes
  • However, styling using inline or internal CSS is permanent and not flexible
  • External CSS is the most common way to style a webpage
  • External CSS allows most developers t keep all their CSS rules in a separate file
  • VOCABULARY: separation of concerns - this refers to separating code into different files based on use
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • When using an external style sheet do not forget to add a link to the HTML in the head section in the metadata.

Git

  • git status: checks what branch we are currently on
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch
  • git commit -m "[description]": this code commits the file and adds a message with what the commit contains
  • git pull: this command receives a branch's modifications in the local environment
  • git push origin: this command pushes the changes made locally to the remote GitHub branch. the origin code indicates the sorice of the pull will be in the GitHub repo.
